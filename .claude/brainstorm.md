# 🧠 Flux Mall Server - 이커머스 프로젝트 브레인스토밍

## 📋 Phase 1: 현재 상태 파악

### ✅ 이미 구현된 도메인
- **회원 (Member)**: 회원가입, 로그인, 프로필 관리
- **인증 (Auth)**: JWT 기반 이중 토큰 인증 시스템
- **상품 (Product)**: 상품 등록, 조회, 검색
- **장바구니 (Cart)**: 장바구니 아이템 관리
- **위시리스트 (Wishlist)**: 찜 상품 관리
- **주문 (Order)**: 주문 생성, 상태 관리
- **리뷰 (Review)**: 상품 리뷰 작성
- **배송지 (Address)**: 배송지 정보 관리
- **최근 본 상품 (RecentView)**: 조회 기록 관리

## 🤔 Phase 2: 탐색적 질문 - 백엔드 포트폴리오 방향성 설정

### 🎯 비즈니스 모델 & 타겟 고객

1. **비즈니스 타겟** ✅ **확정: B2C 입점 마켓플레이스**
   - **타겟**: 다중 판매자 입점 B2C 마켓플레이스 (예: 쿠팡, 11번가 스타일)
   - **카테고리**: 전자제품/패션 위주의 멀티카테고리
   - **백엔드 포인트**: 대용량 트래픽 처리, 다중 테넌트 아키텍처

2. **판매자 구조** ✅ **확정: 다중 판매자 입점 시스템**
   - **판매자**: N명의 셀러가 동시에 입점하여 상품 판매
   - **수수료 정책**: 판매 금액의 N% 수수료 부과
   - **백엔드 포인트**:
     - 분산 트랜잭션 (주문-재고-정산)
     - 정산 배치 처리 (수수료, 배송비 정산)
     - 판매자별 대시보드 (실시간 판매 통계)

### 💰 결제 & 정산 시스템

3. **결제 통합** ✅ **확정: PG사 연동 + 멀티 결제 수단**
   - **결제 수단**: 신용카드, 간편결제 (토스/카카오/네이버), 계좌이체
   - **PG사**: 토스페이먼츠 연동
   - **백엔드 포인트**:
     - 결제 트랜잭션 일관성 (TCC 패턴)
     - 재고 동시성 제어 (Redis 분산 락)
     - 결제 실패 시 롤백 처리

4. **정산 프로세스** ✅ **확정: 배치 기반 일일 정산**
   - **정산 주기**: D+1일 정산 (배치 처리)
   - **정산 항목**: 판매금액 - 수수료 - 배송비 = 실수령액
   - **백엔드 포인트**:
     - Spring Batch를 활용한 대용량 정산 처리
     - 정산 내역 테이블 파티셔닝
     - 정산 오류 처리 및 재시도 메커니즘

### 📦 배송 & 물류

5. **배송 시스템** ✅ **확정: 택배사 API 연동 + 실시간 추적**
   - **택배사**: CJ대한통운, 로젠, 우체국 API 연동
   - **배송비 정책**:
     - 3만원 이상 무료배송
     - 도서산할 3,000원 추가
   - **백엔드 포인트**:
     - 택배사 API 통합 (송장 생성, 추적 조회)
     - 배송 상태 WebSocket 실시간 알림
     - 배송비 캐싱 전략 (지역별 요금)

### 🎁 프로모션 & 마케팅

6. **프로모션 기능** ✅ **확정: 쿠폰 + 포인트 + 한정수량 이벤트**
   - **쿠폰 시스템**:
     - 선착순 쿠폰 (Redis Lua Script로 한정수량 제어)
     - 사용자별 발급 쿠폰 (DB 중복 방지)
   - **포인트 시스템**:
     - 구매 확정 시 적립
     - 포인트 유효기관 관리 (배치 만료 처리)
   - **백엔드 포인트**:
     - Redis를 활용한 한정수량 쿠폰 선점 시스템
     - 포인트 정산 트랜잭션 분리
     - 쿠폰 발급 스케줄러

### 📊 데이터 & 분석

7. **데이터 분석** ✅ **확정: 실시간 대시보드 + 검색엔진**
   - **대시보드**: 매출/주문/방문자 실시간 통계
   - **검색 엔진**: Elasticsearch 도입 (상품 검색, 필터, 자동완성)
   - **추천 시스템**: 최근 본 상품 기반 협업 필터링
   - **백엔드 포인트**:
     - Elasticsearch 인덱싱 전략
     - 실시간 통계 캐싱 (Redis + Caffeine 2레벨 캐시)
     - 검색 로그 수집 및 인기 검색어 집계

## 💡 Phase 3: 백엔드 포트폴리오 핵심 기술 도전 과제

### 🔒 보안 강화 - 인증/인가 고도화

**기술적 포인트:**
- **OAuth2 + Resource Server 구현**: 카카오/네이버 소셜 로그인 연동
- **Redis 기반 Rate Limiting**: API 요청 제한 (사용자/IP별)
- **Refresh Token Rotation**: 보안 강화를 위한 토큰 순환
- **이메일 인증**: 회원가입 시 이메일 확인 (RabbitMQ 비동기 전송)

**포트폴리오 어필 포인트:**
- JWT 토큰 관리 전략
- 분산 환경에서의 인증/인가 처리
- OAuth2 표준 구현 경험

---

### 🚀 성능 최적화 - 대용량 트래픽 처리

**기술적 포인트:**
- **다중 캐싱 전략**:
  - L1: Caffeine (로컬 캐시) - 핫 데이터
  - L2: Redis (분산 캐시) - 공유 데이터
- **캐시 사이드 패턴 적용**: 조회 성능 최적화
- **데이터베이스 파티셔닝**: 대용량 주문/정산 테이블 분리
- **쿼리 튜닝**: N+1 문제 해결, 인덱스 전략

**포트폴리오 어필 포인트:**
- 2레벨 캐시 아키텍처 설계
- 캐시 동기화/무효화 전략
- 대용량 데이터베이스 최적화

### 📦 검색 엔진 - Elasticsearch 도입

**기술적 포인트:**

- **인덱싱 전략**:

  - 역인덱스 구성
- **검색 로그 수집**: 인기 검색어 집계

**포트폴리오 어필 포인트:**
- 검색 엔진 도입 경험
